# Go 编写高性能应用程序的模式和技术

> 原文：[`goperf.dev/`](https://goperf.dev/)

**Go 应用优化指南**是一系列深入的技术文章，旨在帮助开发者从他们的 Go 代码中获得更多性能，而不依赖于猜测或货船文化模式。如果你正在构建需要处理真实负载的服务——API、后端管道或分布式系统——本指南专注于实际生产中真正重要的底层行为和调整机会。

Go 没有给你在 C++或 Rust 中找到的那种细粒度控制，但它确实给你足够的可见性来推理性能——以及足够的工具来采取行动。从理解分配模式、减少 GC 开销到构建高效的网络服务和大规模管理并发，本系列专注于既实用又可衡量的优化。

目标不是编写巧妙的代码——而是编写快速、可预测的代码，在压力下也能保持稳定。本指南中的每一项内容都基于真实案例，去除了理论，旨在让你现在就能应用。

## 性能的常见 Go 模式

这第一篇文章系列涵盖了在编写真实世界 Go 代码时反复出现的性能模式。这不是一个详尽的列表，但它触及了小改动往往能带来明显差异的领域：

+   正确使用 `sync.Pool`

+   减少不必要的分配

+   影响缓存性能的结构布局和内存对齐细节

+   不拖慢快速路径的错误处理

+   使用接口而不付费

+   重复使用切片和在原地排序

每个模式都包括你可以直接应用的真正代码和数字——没有理论，没有冗余。

## Go 中的高性能网络

本节重点探讨了在 Go 中构建快速、可靠的网络服务所需的内容。它不仅涵盖了如何使用标准库，还涵盖了在处理真实负载时如何进一步推动它。

包括以下主题：

+   有效使用 `net/http`、`net.Conn` 和连接池

+   在不崩溃的情况下服务数千个并发连接

+   调整 Go 的调度器和系统级设置（GOMAXPROCS, epoll, kqueue）

+   使用负载 shedding 和 backpressure 构建弹性的服务

+   防止长期连接中的内存泄漏

+   选择和调整传输协议：TCP、HTTP/2、gRPC、QUIC

虽然目标是保持一切基于实际示例，但这一部分由于正在开发的网络主题的探索性，目前更偏向于理论。
